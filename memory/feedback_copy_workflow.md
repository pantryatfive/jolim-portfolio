Always show copy drafts or options in chat first before writing them into the code — especially for case study sections like Overview, Learnings, Design Rationale, and body paragraphs.

**Why:** Jon refines the voice and wants to approve tone before it's live. He caught this on Qatar Airways when I wrote all three options to the page without his sign-off first.

**How to apply:** For new sections or rewrites, present the draft text inline in the conversation. Only commit to code once he says "yes" or picks an option. Exception: scaffolding/placeholder content is fine to write directly.
